03 — Memory

Memory isn't one magic bucket.

It lives in three different places. Mixing them is how agents get confused and hallucinate state. Keep them separate.

Foundation memory

Business truth
  • PostgreSQL
  • Source contracts
  • Decisions log
  • Backlog
  • Change events

Keeps the business accountable. Nothing the system calls "true" should live outside here.

Agent memory

Assistant recall
  • OpenClaw native memory
  • Assistant continuity
  • Cross-session recall
  • Personal assistant context

Turns on only after the Foundation baseline is proven. Premature "smart memory" is noise.

Workspace identity

Session context
  • AGENTS.md
  • SOUL.md
  • USER.md
  • Local repo instructions

Tells a session how to behave inside a workspace. Not the same thing as business memory.

04 — The Audit

What's solid. What to watch.

Solid
Layer separation is architecturally sound

Foundation → Runtime → Identity → Execution → Model is a proven pattern, not invented. Siri and Cortana route the same way.

Solid
Foundation-first discipline holds

Not pivoting mid-rebuild is the correct call, especially given BCrew-Buddy's "data before agents" lesson.

Solid
Honest naming of current Harlan

"OpenClaw's Telegram bot mounted to this repo, wearing Harlan's name" is truth-telling, not pretending identity is clean.

Solid
Memory layering is accurate
