# Documentation Authority Map

Status: Active
Last updated: 2026-04-25

This repo has a lot of documentation on purpose. Some files are active truth. Most are evidence.

Use this map so old handoffs and audits do not compete with the current system.

For the full markdown inventory, use [`INDEX.md`](INDEX.md). It is generated by `node scripts/generate-doc-indexes.mjs` and classifies every doc as active truth, supporting truth, evidence, or reconciliation work.

## Evidence

- [`handoffs/`](handoffs/) — session checkpoints, source validations, and reconstructed transcripts
- [`audits/`](audits/) — historical and current audit artifacts
- [`rebuild/plan-history/`](rebuild/plan-history/) — retired rebuild plans and decision snapshots kept for traceability, not current doctrine
- [`research/`](research/) — research notes and older doctrine debates
- [`superpowers/`](superpowers/) — older design work and implementation plans
- [`decks/`](decks/) — presentation assets

Evidence files can be useful, but they are not current doctrine unless the active truth set links to or promotes them.

## Rule

If a claim appears only in a handoff, old audit, or research note, treat it as evidence. Promote it into active docs or the DB-backed backlog before building from it.
