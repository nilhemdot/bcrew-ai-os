# Strategy / Action Loop Correction

Date: 2026-04-24
Status: doctrine correction

## Correction

Steve explicitly rejected the "daily brief" framing.

The goal is Foundation first:

- source contracts and trusted connections
- extraction with provenance, sensitivity, and source evidence
- atoms, decisions, tasks, contradictions, and source-backed operating facts
- synthesis that ranks what matters
- routed actions with owner, reason, evidence, and back-link
- completed actions and decisions flowing back into the operating record

The first useful hub is Strategy Hub, not an email digest or daily dashboard brief.

## Correct Loop

```text
source -> atom/candidate -> synthesized item -> routed action/decision/contradiction -> completion/resolution -> source-backed memory
```

If a synthesized item does not become a decision, contradiction, task, routed action, or explicit ignore/snooze decision, the loop is incomplete.

## Runtime Implication

Extraction miners still matter because Strategy Hub needs fresh evidence.

Synthesis still matters, but it is a function inside the operating loop, not the product.

Scheduling a digest is not the milestone. The milestone is:

- useful intelligence lands in the right operating lane
- the right owner gets a reason and source evidence
- Steve can make or delegate a decision
- the system records the outcome so the same issue is not rediscovered forever

## Next Build Lens

Review future work against this question:

> Does this move source-backed intelligence toward a real decision, contradiction, task, or owner-bound action?

If not, it is probably infrastructure only and should be named as such.
