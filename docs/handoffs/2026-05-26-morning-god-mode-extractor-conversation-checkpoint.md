## Core Correction

Steve is not asking for a weak page reader, a manual Codex browsing session, or a no-click/no-navigation extractor.

God Mode extractor means the product/system can do source work like a capable human within approved boundaries:

- Read text and metadata.
- Watch/listen/see video and page visuals.
- Navigate and click through relevant allowed source surfaces.
- Follow safe public/free links.
- Inspect resources, repos, docs, newsletters, free communities, classes/courses/classrooms, and creator source stacks.
- Capture and package safe free resources/value.
- Evaluate paid gates and tell Steve whether a purchase/course/community looks worth buying.
- Stop at real boundaries: purchase/checkout/payment, paid/private/member walls, login/auth/session gaps, MFA, unapproved forms/signups, unsafe downloads, posting/commenting/messaging, account/profile/credential mutation, or external writes.

Manual browser work by Codex is not proof. The system runner must perform the work and produce artifacts/evidence.

## What Steve Explicitly Wants Built

The extractor is the full reusable system. Each source family needs its own SOP on top of the shared runtime.

Shared runtime layers:

- Eyes: screenshots/visual classification, preferably with Gemini eyes where useful.
- Ears: audio/video understanding for video sources.
- Read: DOM/text/transcript/file/page extraction.
- Hands: governed browser navigation and clicks.
- Brain: source value evaluation, link/resource classification, synthesis, and grading.
- Evidence: artifacts, screenshots/keyframes, source URLs, timestamps, link dispositions, stop reasons.
- Boundaries: paid/private/auth/forms/downloads/write/mutation stops.
- Output: creator/source cards, build candidates, source value packages, Director/Scoper inputs.
- Grade: creator/source grading by lane, not one global grade.
- Autopilot: runs without Steve babysitting after proof.

## YouTube SOP

YouTube is not just video watching.

Required flow:

1. Inventory all approved creators, not only the first 10 displayed rows.
2. Show every creator on Dev source cards with source URL, source stack, lane grades, and why/rank explanation.
3. Triage metadata first: title, description, creator, recency, relevance, duration/risk, source grade, and links.
4. Watch a baseline of the first/latest 10 relevant public videos per creator.
5. Full watch means video/audio/visual review, not transcript-only.
6. Extract the whole YouTube page and description.
7. Follow safe public/free resource links from descriptions/pages.
8. Auto-read public GitHub/docs/repos/resource pages where safe.
9. Detect newsletters/free resources/free communities linked from the video.
10. For free resources, capture/package the value where safe.
11. For free communities, route into the free-community SOP.
12. For paid gates, stop and produce a buy/not-buy value evaluation for Steve.
13. Generate build ideas, source value notes, and evidence.
14. Rank/grade creators by lane after enough evidence.
15. Deepen S/A creators, selectively use B, park/throttle weak C/D creators.
16. Update morning autopilot recommendations and Dev cards.

Steve does not want broad YouTube spend before the extractor can do the full source job. Video-only catch-up is not God Mode.

## Paid/Auth Source Rule

Paid/auth sources should be closed-loop, not Steve-babysat.

Gold standard:

- system uses approved credential/session broker when available
- system uses source-specific login recipe

- system waits/resumes/fails closed
- system records auth-needed artifacts and next action
