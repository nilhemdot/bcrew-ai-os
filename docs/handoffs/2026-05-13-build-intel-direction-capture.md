Status: context capture and sprint-planning input. This is not an approved implementation sprint.

## Core Decision

Build Intel sources are for improving AIOS itself. They are dev/build intelligence, not marketing content inputs.

The near-term goal is not to rebuild the old autonomous dev team. The goal is a governed Foundation lane where approved external builder content can become source-backed observations and proposal-only card enrichments.

## Steve Direction Captured

- No autonomous dev. Advisors may propose; Steve and Codex approve; Codex builds one governed slice at a time.
- Default to code. Use LLM/agent behavior only for natural-language understanding, synthesis, generation, scoring, or judgment.
- Public YouTube should not default to Playwright screenshot scraping. Prefer official/permitted discovery and video-understanding routes first; screenshots/keyframes require explicit use/storage policy.
- Playwright/browser extraction belongs behind governed account/session preflight, mainly for authorized paid apps, course portals, private workflows, or fallback visual proof.
- Build Intel and marketing-content production stay separate.
- Paid/auth sources such as Skool, myICOR, and Loom require Steve-approved access and content-use boundaries before extraction.
