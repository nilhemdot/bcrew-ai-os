# Marketing Avatar Registry

Status: governed source-backed registry for `AVATAR-IMPORT-001`
Imported source: `docs/marketing/avatars/source/old-bcrew-buddy/`

This folder carries forward the old BCrew-Buddy avatar work without starting the full marketing pipeline.

## Registry Shape

- `RETAIN-*` avatars are client-side buyer/seller personas.
- `ATTRACT-*` avatars are agent recruiting personas.
- Registry IDs are stable references for future atoms, briefs, Scoper output, and marketing cards.
- Avatars are optional overlays on top of intelligence atoms. Non-marketing atoms should not be forced to carry avatar IDs.

## Source Files

- `source/old-bcrew-buddy/retain-avatars.md` keeps the full 10 RETAIN client profiles.
- `source/old-bcrew-buddy/attract-avatars.md` keeps the full 5 ATTRACT agent profiles.
- `source/old-bcrew-buddy/avatar-reference-brief.md` is the concise registry seed.
- `source/old-bcrew-buddy/README.md` is the old-system usage note.

## Boundary

This import makes the avatar data governed and referenceable. It does not create campaigns, content calendars, writers, editors, designers, schedulers, Brand Guardian enforcement, or monthly refresh automation.
